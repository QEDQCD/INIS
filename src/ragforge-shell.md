# ragforge-shell 模块技术总结

## 模块定位
`ragforge-shell/` 是命令行工具子模块目录，用于通过 CLI 管理数据集、文档上传、解析和检索流程。

## 当前状态
- 仓库中该目录以 Git 子模块方式引用。
- 当前工作区仅保留子模块入口，不包含完整源码。

## 预期职责
- 提供系统状态检查、数据集管理、文档上传、文档解析和检索问答命令。
- 适合作为自动化脚本、测试脚本和运维排障的轻量入口。

## 与主仓库关系
- `rag/README.md` 中已有面向 CLI 的使用示例。
- `tests/` 中存在与 CLI 相关的验证脚本，说明该子模块被视为正式接入链路的一部分。
