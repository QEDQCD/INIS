# agent 模块技术总结

## 模块定位
`agent/` 用于承载图式工作流与 Agent DSL，适合把检索、推理、工具调用和业务节点编排成可复用流程。

## 目录概览
- `component/`：工作流节点或组件定义。
- `templates/`：模板化 DSL 或场景样例。
- `test/`：DSL 示例与调试客户端。

## 核心能力
- 用节点与边表达复杂执行链路，而不是只支持线性 DAG。
- 提供 DSL 样例，便于快速验证 Agent 编排思路。
- 可与主服务联动，通过租户与知识库上下文运行检索组件。

## 关键使用方式
- 启动主服务后，可用 `agent/test/client.py` 测试 DSL。
- 典型输入包括 DSL 文件、租户标识与是否流式输出等参数。

## 工程价值
- 让复杂问答、检索、工具协作逻辑从代码中抽离出来。
- 为后续引入更多节点类型和模板提供稳定扩展点。

## 注意事项
- 运行前需要准备可用的主服务配置与知识库上下文。
- DSL 中的知识库、模型、工具参数都应使用环境专属占位值，不应写入真实敏感数据。
