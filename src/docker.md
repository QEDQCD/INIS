# docker 模块技术总结

## 模块定位
`docker/` 提供本地开发、标准部署、GPU/NPU 变体和配套基础服务编排，是平台的主要容器化交付目录。

## 主要资产
- `docker-compose-base.yml`：源码运行后端时所需基础依赖。
- `docker-compose.yml`、`docker-compose-dev.yml`：完整部署与开发环境编排。
- `docker-compose-gpu*.yml`、`docker-compose-npu.yml`：异构硬件部署变体。
- `nginx/`、`certs/`：网关和证书相关资源。
- `start.sh`：交互式启动脚本。

## 支撑能力
- 一次性拉起 API、Web、数据库、缓存、搜索与对象存储。
- 支持生产与开发环境切换。
- 支持不同计算环境和地区化镜像方案。

## 技术特点
- 将基础设施依赖显式化，降低本地搭建门槛。
- 通过多套 Compose 文件适配不同部署形态。
- 适合与 `conf/` 联动完成多环境参数切换。

## 使用建议
- 真实部署时不要沿用示例账号、示例口令和示例主机。
- 建议把环境变量与配置模板分离管理，保证镜像与仓库无敏感信息。
