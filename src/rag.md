# rag 模块技术总结

## 模块定位
`rag/` 是平台的核心处理链路，负责文档切片、索引、检索、重排、问答生成和任务执行。

## 目录概览
- `app/`：按文档类型组织的解析应用层，如论文、书籍、表格、图片、问答等。
- `llm/`：模型适配与调用封装。
- `nlp/`：分词、检索与自然语言处理能力。
- `svr/`：任务执行器和后台服务。
- `utils/`：缓存、存储、连接器与通用工具。

## 核心职责
- 对不同类型文档选择合适解析器。
- 构建分块、标签、关键词与索引内容。
- 通过向量、关键词和图谱等多路方式进行召回。
- 把检索结果组织成可供模型回答的上下文。

## 关键实现点
- `svr/task_executor.py` 是解析任务主入口。
- `app/*` 体现不同文档体裁的专用处理策略。
- `llm/` 和 `utils/` 负责把模型、缓存和存储接入统一流程。

## 平台意义
- 这是整个仓库的业务中枢，其他模块多数围绕它提供输入、增强或交付能力。
