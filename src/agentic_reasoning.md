# agentic_reasoning 模块技术总结

## 模块定位
`agentic_reasoning/` 实现多步检索式推理，核心目标是在回答前先生成搜索意图、逐步补充证据，再汇总输出。

## 主要文件
- `deep_research.py`：深度研究主流程，串联推理、查询抽取、检索与结果整合。
- `prompts.py`：定义搜索标签、提示模板和最大搜索轮次。

## 核心流程
1. 根据用户问题生成阶段性推理文本。
2. 从推理文本中抽取待搜索问题。
3. 联合知识库检索、图谱检索和可选 Web 检索获取证据。
4. 把新证据写回推理上下文，进入下一轮搜索。
5. 达到上限或信息充分后结束。

## 技术特点
- 支持多轮检索，不依赖单次召回。
- 可按配置挂接 Tavily 等外部检索源。
- 通过标签协议约束模型输出，便于程序化解析。

## 适用场景
- 开放域问题求解。
- 需要多跳证据整合的复杂问答。
- 需要把知识库、图谱和外部信息统一纳入推理链路的场景。
