# tests 模块技术总结

## 模块定位
`tests/` 用于验证接口、模型接入、文档解析、数据库兼容性和脚本行为，是项目质量保障入口。

## 测试内容分布
- `dm_database/`：达梦数据库适配测试。
- `openai/`：模型接口与协议验证。
- `pdf_parser/`：PDF 解析实验与比对。
- `test_models/`：模型配置、示例输入与生成脚本。
- 根目录脚本：接口调用、上传解析、登录、配置和用户流程测试。

## 特点
- 既有单元风格脚本，也有偏集成验证的端到端测试。
- 覆盖文档上传、解析触发、模型连接、配置加载等高风险链路。
- 适合在功能变更后做定向回归。

## 使用建议
- 优先运行与改动模块最相关的测试。
- 涉及模型、存储、数据库的测试前，应先准备独立测试环境和脱敏配置。
